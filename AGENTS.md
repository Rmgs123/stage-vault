Перед началом любой работы прочитай два файла:
1. SPEC.md — полная спецификация проекта (архитектура, user stories, API, схема БД)
2. TASKS.md — текущий трекер задач. После выполнения задачи обнови её статус (🔲 → ✅). Может не понадобиться если ты делаешь какую-то другую задачу или исправляешь баг.

3. design/references/ — примеры кода UI-компонентов для референса стиля. Используй их как образец визуального стиля (цвета, скругления, отступы, шрифты) при создании новых компонентов.
4. design/screens/ — скриншоты финального дизайна экранов. Сверяйся с ними при вёрстке.
5. Также обновляй и другие .md файлы по типу TESTING.md и так далее, если ты что-то менял что могло затронуть их содержимое.
