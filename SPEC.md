# StageVault — Спецификация проекта

> Облачная платформа управления контентом мероприятий с ИИ-ассистентом
> Домен: stage-vault.ru | Репозиторий: [github.com/.../stage-vault](https://github.com/Rmgs123/stage-vault)

---

## 1. Обзор проекта

### 1.1 Что это

StageVault — веб-приложение, которое позволяет организаторам мероприятий собрать весь контент (музыку, презентации, сценарии, документы, изображения и любые другие файлы) в одном месте, получить к нему доступ на площадке по короткому коду без авторизации, работать над мероприятием в команде и использовать ИИ-ассистента для анализа и проверки контента.

### 1.2 Проблема

Организаторы офлайн-мероприятий (квизы, лекции, концерты, культурные вечера, образовательные занятия) сталкиваются с типичными проблемами:

- Контент разбросан по разным сервисам: музыка на флешке, презентация в Google Drive, сценарий в мессенджере
- На площадке нужно входить в личные аккаунты на чужом компьютере — это неудобно и небезопасно
- Флешки не читаются, файлы забыты, VPN для облака не работает
- Нет единого интерфейса для проведения: приходится переключаться между окнами, вкладками, приложениями
- Нет автоматической проверки контента (опечатки, дубликаты, ошибки)
- Нет удобного инструмента для совместной подготовки мероприятия в команде

### 1.3 Решение

StageVault предоставляет:

- Единое хранилище всех файлов мероприятия с категоризацией
- Доступ к мероприятию на любом компьютере по 6-символьному коду без авторизации
- Полноценный пульт ведущего: плеер, презентация на весь экран, таймлайн-сценарий
- Командную работу: приглашение участников, роли, совместное редактирование
- ИИ-ассистент с доступом ко всем файлам проекта для анализа и проверки
- Возможность редактирования контента прямо на площадке (добавить файл с флешки, поправить текст)

### 1.4 Целевая аудитория

- Организаторы квизов и интеллектуальных игр
- Ведущие культурных и образовательных мероприятий
- Молодёжные организации и творческие инициативы
- Преподаватели и лекторы
- Городские культурные и просветительские проекты

### 1.5 Конкурентное преимущество

Ни одно существующее решение не объединяет: хранение медиа + быстрый доступ по коду + интерфейс проведения + ИИ-анализ контента + командную работу. Облачные диски хранят файлы, но не дают интерфейс проведения. Платформы мероприятий ориентированы на онлайн/гибрид и стоят дорого. Квиз-платформы решают только задачу викторины.

## 2. Пользователи и роли

### 2.1 Роли в системе

| Роль | Описание | Как получает |
|------|----------|-------------|
| **Владелец** | Создатель проекта мероприятия. Полный доступ: управление файлами, сценарием, участниками, ролями. Может приглашать, удалять участников, генерировать коды доступа. Роль неизменяема. | Автоматически при создании проекта |
| **Редактор** | Член команды. Может загружать, редактировать, удалять файлы, работать со сценарием. Не может управлять участниками и ролями. | Назначается владельцем |
| **Зритель** | Член команды. Может просматривать и скачивать файлы. Не может изменять контент. | Роль по умолчанию при приглашении |
| **Гость по коду** | Временный доступ по 6-символьному коду. Получает тот же интерфейс и возможности, что и авторизованный пользователь-владелец (включая редактирование), но: код временный (ограничен по TTL), сессия не привязана к аккаунту, при закрытии вкладки/истечении TTL — доступ теряется. | Генерируется владельцем или редактором |

### 2.2 Приглашение в команду

- Владелец приглашает участников **по никнейму** или **по email** (поиск по зарегистрированным пользователям)
- Приглашение появляется **внутри приложения** в Inbox приглашённого пользователя: «Вас пригласил X в проект Y»
- Приглашённый может **принять** или **отклонить**
- Email для приглашения **не отправляется** — только in-app уведомление
- По умолчанию приглашённый получает роль **Зритель**
- Владелец может изменить роль в окне управления участниками

## 3. Функциональные модули

### 3.1 Аутентификация и профиль

#### Регистрация
- Поля: email, пароль, подтверждение пароля
- После регистрации — письмо на email с ссылкой подтверждения
- Аккаунт активен только после подтверждения email

#### Вход
- Email + пароль
- «Забыли пароль?» — письмо со ссылкой для сброса

#### Профиль
- Отображение email (нередактируемый)
- Никнейм (редактируемый, уникальный, виден команде)
- Смена пароля (через подтверждение на email)
- Переключение темы (светлая / тёмная)

### 3.2 Проекты мероприятий (Dashboard)

#### Список проектов
- Карточки мероприятий с названием, датой, обложкой, статусом
- Фильтр: мои / в которых я участник
- Сортировка: по дате, по названию
- Быстрое создание нового проекта

#### Создание / редактирование проекта
- Название
- Дата и время проведения
- Описание (опционально)
- Обложка (опционально, изображение)
- Статус: черновик / готов / завершён

### 3.3 Хранилище файлов

#### Принцип: храним всё, показываем умно

Все файлы хранятся в S3-совместимом хранилище. Каждый файл принадлежит конкретному проекту мероприятия.

#### Категории (автоматические вкладки в интерфейсе):

| Категория | Расширения | Возможности в интерфейсе |
|-----------|-----------|-------------------------|
| **Музыка** | mp3, wav, ogg, flac, aac, m4a | Встроенный аудиоплеер, плейлист, порядок воспроизведения, управление громкостью |
| **Презентации** | pptx, pdf | Просмотр слайдов, полноэкранный режим для проектора |
| **Изображения** | jpg, jpeg, png, gif, svg, webp | Галерея, предпросмотр, полноэкранный показ |
| **Видео** | mp4, webm, mov | Встроенный видеоплеер |
| **Документы** | txt, md, doc, docx, rtf | Текстовый просмотр (txt/md встроенный, остальные — скачивание) |
| **Прочее** | Все остальные расширения | Хранение, скачивание, метаданные (имя, размер, дата загрузки) |

#### Действия с файлами:
- Загрузка (drag-and-drop + кнопка, в том числе множественная)
- Скачивание
- Удаление
- Переименование
- Добавление заметки/описания к файлу
- Изменение порядка (для музыки)
- Автоматическая категоризация по расширению

#### Ограничения (настраиваемые):
- Максимальный размер одного файла: 500 MB (по умолчанию)
- Максимальный объём на проект: 5 GB (по умолчанию)

### 3.4 Сценарий мероприятия (Timeline)

Каждый проект имеет один сценарий — это упорядоченный список блоков.

#### Блок сценария:
- **Название** — краткое описание этапа (например: «Вступительное слово», «Раунд 1», «Музыкальная пауза»)
- **Описание** — развёрнутый текст (подсказки ведущему, тезисы)
- **Планируемая длительность** — в минутах
- **Таймер** — обратный отсчёт, который можно запустить/остановить/сбросить
- **Чекбокс** — «выполнено» (для отслеживания хода мероприятия)
- **Прикреплённые файлы** — ссылки на файлы из хранилища проекта (например, к блоку «Раунд 1» прикреплена презентация с вопросами и фоновая музыка)

#### Действия:
- Добавление / удаление / перестановка блоков (drag-and-drop)
- Редактирование содержимого блока
- Запуск таймера на текущем блоке
- Отметка блока как выполненного
- Общий прогресс мероприятия (прогресс-бар по чекбоксам)

### 3.5 Режим площадки (доступ по коду)

#### Генерация кода:
- Владелец или редактор генерирует код в настройках проекта
- Код: 6 символов, буквенно-цифровой, регистронезависимый
- TTL: задаётся при генерации (по умолчанию: 24 часа)
- Можно отозвать код досрочно
- Одновременно может быть активен только один код на проект

#### Вход по коду:
- Отдельная страница: `stage-vault.ru/go`
- Одно поле ввода кода + кнопка «Войти»
- При валидном коде — сервер выдаёт временный JWT-токен с ограниченным TTL
- Открывается полный интерфейс проекта (тот же, что у владельца)

#### Возможности в режиме площадки:
- Все функции пульта ведущего: плеер, презентация, таймлайн, таймеры
- Редактирование: добавление файлов (в том числе с флешки/компьютера), правка текстов в сценарии
- ИИ-ассистент
- Просмотр и скачивание всех файлов

#### Ограничения:
- Нельзя управлять участниками и ролями
- Нельзя удалять проект
- Нельзя генерировать новые коды
- Сессия завершается по истечении TTL токена

### 3.6 Пульт ведущего (Presenter View)

Это не отдельная страница, а **режим отображения** в рамках проекта. Доступен и через обычную авторизацию, и через код.

#### Компоненты пульта:

**Левая панель — Сценарий (Timeline):**
- Список блоков с чекбоксами
- Текущий блок подсвечен
- Таймер текущего блока
- Общий прогресс

**Центральная область — Контент:**
- Если к блоку прикреплена презентация — показ слайдов с навигацией
- Если прикреплено изображение — полноэкранный показ
- Кнопка «На проектор» (Fullscreen API) — выводит центральную область на полный экран

**Нижняя панель — Аудиоплеер:**
- Плейлист мероприятия (все файлы из категории «Музыка»)
- Play / Pause / Next / Previous / Громкость
- Текущий трек и прогресс

**Правая панель (сворачиваемая) — Быстрый доступ:**
- Файлы проекта по категориям
- ИИ-ассистент (чат)

### 3.7 ИИ-ассистент

#### Интерфейс:
- Отдельная сворачиваемая панель (чат) внутри проекта
- Поле ввода сообщения + история диалога
- Доступен и в обычном режиме, и в режиме площадки

#### Возможности (v1 — только чтение и анализ):
- **Видит** все файлы проекта: их названия, типы, содержимое текстовых файлов, метаданные
- **Видит** сценарий: все блоки, описания, тайминги
- **Может отвечать** на вопросы по контенту: «Сколько вопросов в презентации?», «Есть ли дубликаты?», «Какая общая длительность сценария?»
- **Может указывать** на ошибки: опечатки в текстах, подозрительные ответы в квизах, слишком длинные/короткие блоки сценария
- **Не может** редактировать файлы или сценарий

#### Техническая реализация:
- Бэкенд выступает прокси к LLM API (ключи хранятся на сервере, клиент не имеет прямого доступа)
- Перед отправкой запроса бэкенд собирает контекст: список файлов, содержимое текстовых файлов, структуру сценария
- Модель: настраиваемая через конфигурацию (OpenAI / Anthropic / self-hosted)

### 3.8 Inbox (Уведомления)

- Иконка с бейджем в шапке приложения
- Типы уведомлений:
  - Приглашение в проект (с кнопками «Принять» / «Отклонить»)
  - Изменение роли в проекте
  - Удаление из проекта
- Отметка как прочитанное
- Пока без email-уведомлений — только in-app

### 3.9 Заглушка для мобильных устройств

- При ширине экрана < 1024px показывается полноэкранная заглушка:
  - Логотип StageVault
  - Текст: «StageVault пока не поддерживает мобильные устройства. Пожалуйста, откройте приложение на компьютере.»
  - Ссылка на stage-vault.ru
- Никакой функциональности на мобильных устройствах не предоставляется

### 5.2 Стек технологий

| Слой | Технология | Обоснование |
|------|-----------|-------------|
| **Frontend** | React 18+, TypeScript, Tailwind CSS, React Router, Zustand (state) | Широкая экосистема, типизация, быстрый UI |
| **Backend** | Node.js, Fastify, TypeScript | Один язык на всём стеке, высокая производительность |
| **ORM** | Prisma | Типобезопасность, миграции, удобная работа с PostgreSQL |
| **БД** | PostgreSQL | Надёжность, JSON-поля для метаданных, полнотекстовый поиск |
| **Файлы** | MinIO (S3-совместимое) | Самохостинг, S3 API, не зависим от зарубежных облаков |
| **Аудиоплеер** | Howler.js или встроенный HTML5 Audio | Кроссбраузерность, плейлисты |
| **PDF/PPTX** | pdf.js (для PDF), LibreOffice конвертация PPTX → PDF (серверная) | Встроенный просмотр без внешних сервисов |
| **ИИ** | REST API к LLM (настраиваемый провайдер) | Гибкость, возможность замены модели |
| **Email** | Nodemailer + SMTP | Подтверждение email, сброс пароля |
| **Auth** | JWT (access + refresh tokens) | Stateless, подходит для кодов площадки |
| **Деплой** | Docker, docker-compose, Nginx | Один `docker-compose up` поднимает всё |

### 5.4 Схема базы данных (Prisma)

```prisma
model User {
  id            String   @id @default(uuid())
  email         String   @unique
  passwordHash  String
  nickname      String?  @unique
  emailVerified Boolean  @default(false)
  theme         String   @default("dark") // "light" | "dark"
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt

  ownedEvents   Event[]         @relation("EventOwner")
  memberships   EventMember[]
  notifications Notification[]
}

model Event {
  id          String   @id @default(uuid())
  title       String
  description String?
  date        DateTime?
  coverUrl    String?
  status      String   @default("draft") // "draft" | "ready" | "done"
  ownerId     String
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  owner       User           @relation("EventOwner", fields: [ownerId], references: [id])
  members     EventMember[]
  files       File[]
  blocks      TimelineBlock[]
  accessCodes AccessCode[]
}

model EventMember {
  id      String @id @default(uuid())
  eventId String
  userId  String
  role    String @default("viewer") // "editor" | "viewer"

  event   Event @relation(fields: [eventId], references: [id], onDelete: Cascade)
  user    User  @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([eventId, userId])
}

model File {
  id          String   @id @default(uuid())
  eventId     String
  name        String
  originalName String
  mimeType    String
  size        Int
  s3Key       String
  category    String   // "music" | "presentation" | "image" | "video" | "document" | "other"
  sortOrder   Int      @default(0)
  note        String?
  createdAt   DateTime @default(now())

  event       Event            @relation(fields: [eventId], references: [id], onDelete: Cascade)
  attachments BlockAttachment[]
}

model TimelineBlock {
  id          String   @id @default(uuid())
  eventId     String
  title       String
  description String?
  durationMin Int?             // Планируемая длительность в минутах
  sortOrder   Int      @default(0)
  completed   Boolean  @default(false)
  createdAt   DateTime @default(now())

  event       Event            @relation(fields: [eventId], references: [id], onDelete: Cascade)
  attachments BlockAttachment[]
}

model BlockAttachment {
  id      String @id @default(uuid())
  blockId String
  fileId  String

  block   TimelineBlock @relation(fields: [blockId], references: [id], onDelete: Cascade)
  file    File          @relation(fields: [fileId], references: [id], onDelete: Cascade)

  @@unique([blockId, fileId])
}

model AccessCode {
  id        String   @id @default(uuid())
  eventId   String
  code      String   @unique
  expiresAt DateTime
  revoked   Boolean  @default(false)
  createdAt DateTime @default(now())

  event     Event @relation(fields: [eventId], references: [id], onDelete: Cascade)
}

model Notification {
  id        String   @id @default(uuid())
  userId    String
  type      String   // "invite" | "role_change" | "removed"
  payload   Json     // { eventId, eventTitle, fromUser, role, ... }
  read      Boolean  @default(false)
  createdAt DateTime @default(now())

  user      User @relation(fields: [userId], references: [id], onDelete: Cascade)
}
```

## 8. Принципы разработки

### 8.1 Код

- Весь код на TypeScript (strict mode)
- ESLint + Prettier с единой конфигурацией
- Каждый модуль бэкенда — отдельная папка со своими router, controller, service, types
- Никаких `any` — строгая типизация
- Все API-ответы в едином формате: `{ data, error, message }`
- Все ошибки обрабатываются централизованно через error middleware

### 8.2 Безопасность

- Пароли хэшируются через bcrypt (salt rounds: 12)
- JWT: access token (15 мин) + refresh token (7 дней)
- Коды площадки: отдельный JWT с укороченным TTL
- Все файлы доступны только через presigned URLs (не прямые ссылки на S3)
- Rate limiting на auth-эндпоинтах
- Валидация всех входных данных (Zod)
- CORS настроен только на конкретный домен

### 8.3 Расширяемость

Архитектура проектируется с учётом будущих расширений:

- **Квиз-модуль**: новый роутер `/api/events/:id/quiz`, новая таблица `Quiz` / `Question` / `Answer`, WebSocket через Socket.io для реалтайм-ответов участников, новый React-роут. Ядро не затрагивается.
- **Командная работа в реальном времени**: добавление Socket.io для синхронизации состояния Timeline между несколькими клиентами.
- **Мобильное приложение**: API уже готов, нужен только мобильный клиент.
- **Мультиязычность (i18n)**: все строки интерфейса выносить в константы/объекты с первого дня, чтобы потом легко заменить на i18n-библиотеку.
- **Шаблоны мероприятий**: дублирование проекта со всеми файлами и сценарием.
